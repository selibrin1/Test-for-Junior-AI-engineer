# Financial AI Assistant

Прототип AI-ассистента для анализа финансовых данных компании (2005–2024).
Ассистент загружает структурированные финансовые показатели, рассчитывает ключевые метрики
и отвечает на аналитические вопросы, используя Perplexity Sonar API.

---

## Установка и запуск

### 1. Клонировать репозиторий

    git clone https://github.com/selibrin1/Test-for-Junior-AI-engineer.git
    cd Test-for-Junior-AI-engineer

### 2. Создать виртуальное окружение и установить зависимости

    python -m venv venv
    source venv/bin/activate        # macOS / Linux
    # venv\Scripts\activate         # Windows

    pip install -r requirements.txt

### 3. Настроить API-ключ

    cp .env.example .env

Открыть `.env` и вставить свой Perplexity API-ключ:

    PERPLEXITY_API_KEY=your_perplexity_api_key_here

Ключ можно получить на https://www.perplexity.ai/settings/api

### 4. Запустить ассистента

    python main.py

Дополнительные флаги:

| Флаг | По умолчанию | Описание |
|------|-------------|----------|
| `--model` | `sonar` | Модель Perplexity (`sonar`, `sonar-pro` и др.) |
| `--data` | `financial_data.csv` | Путь к CSV-файлу с данными |

Пример с другой моделью:

    python main.py --model sonar-pro

## Архитектура

    .
    +-- financial_data.csv      # Исходные данные (20 лет: 2005-2024)
    +-- financial_analyzer.py   # Загрузка CSV и расчёт финансовых метрик
    +-- assistant.py            # Интеграция Perplexity Sonar API + промпт-инжиниринг
    +-- main.py                 # CLI-интерфейс (REPL)
    +-- requirements.txt        # Зависимости Python
    +-- .env.example            # Шаблон переменных окружения

### Описание модулей

**`financial_analyzer.py`**

Читает `financial_data.csv` стандартными средствами Python (`csv.DictReader`) и для каждого года
рассчитывает производные метрики:

- Revenue Growth = (Revenue_t - Revenue_{t-1}) / Revenue_{t-1} * 100
- Operating Income = Revenue - COGS - Operating Expenses
- Operating Margin = Operating Income / Revenue * 100
- Net Margin = Net Income / Revenue * 100

Функция `build_context()` форматирует все данные в текстовую таблицу ASCII, которая дословно
вставляется в системный промпт — модель видит только реальные числа из датасета.

**`assistant.py`**

Обёртка над OpenAI-совместимым клиентом, направленным на `api.perplexity.ai`.

Ключевые решения по промпт-инжинирингу:
- Системный промпт содержит полную таблицу данных+метрик и устанавливает жёсткие правила:
  не придумывать числа, показывать расчёты, отвечать на языке пользователя.
- Температура 0.2 — снижает «творческие» отклонения от данных.
- История диалога сохраняется между вопросами (multi-turn).

**`main.py`**

Интерактивный REPL-интерфейс. Специальные команды: `/reset` — очистить историю, `/quit` — выход.

## Примеры запросов и ответов

### 1. Самый быстрый рост выручки

**Вопрос:**

> В каком году был самый быстрый рост выручки?

**Ответ:**

> Самый быстрый рост выручки наблюдался в **2006 году**.
>
> Расчёт: (155 000 − 120 000) / 120 000 × 100 = **+29.2%**
>
> Это максимальное значение Revenue Growth за весь период 2005–2024.

---

### 2. Анализ трендов прибыльности

**Вопрос:**

> Как изменялась прибыльность компании со временем?

**Ответ:**

> На основе данных за 2005–2024:
>
> **Операционная маржа** держится на стабильном уровне ~25% на протяжении всего периода
> (минимум 24.9% в 2024, максимум 25.0% в большинстве лет).
>
> **Чистая маржа** также стабильна: ~25% (например, 25.0% в 2005, 25.0% в 2024).
>
> Вывод: прибыльность компании практически не изменилась за 20 лет — структура затрат
> масштабировалась пропорционально росту выручки.

---

### 3. Объяснение операционной маржи

**Вопрос:**

> Объясните динамику операционной маржи компании.

**Ответ:**

> Операционная маржа рассчитывается как:
>   Operating Margin = (Revenue − COGS − Operating Expenses) / Revenue × 100
>
> В 2005 году: (120 000 − 50 000 − 40 000) / 120 000 × 100 = 25.0%
> В 2024 году: (3 650 000 − 1 533 000 − 1 205 000) / 3 650 000 × 100 = 24.99%
>
> На протяжении всех 20 лет показатель практически не отклонялся от 25%.
> Это говорит о высокой операционной предсказуемости и масштабируемости бизнеса.

### 4. Уточняющий вопрос в рамках диалога

**Вопрос:**

> А в каком году выручка впервые превысила 1 миллион долларов?

**Ответ:**

> Согласно данным:
> - 2016: Revenue = 1 010 000 $
> - 2015: Revenue = 850 000 $
>
> Выручка впервые превысила 1 000 000 $ в **2016 году** (1 010 000 $).
